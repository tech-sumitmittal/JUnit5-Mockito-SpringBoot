# JUnit5-Mockito-SpringBoot API Endpoints

This README provides a comprehensive list of REST API endpoints for managing `Customer` and `Account` entities in the JUnit5-Mockito-SpringBoot application. All endpoints are accessible via `http://localhost:8080/api`.

---

### To access H2 database from browser
```
http://localhost:8080/h2-console
```

---

## Customer Endpoints

### 1. Create a Customer
```
POST /customers
```
**Request Body:**
```json
{
  "name": "Alice Smith",
  "email": "alice@example.com",
  "mobileNumber": "9123456780"
}
```
**Response:**
```json
{
  "customerId": 1234567890,
  "name": "Alice Smith",
  "email": "alice@example.com",
  "mobileNumber": "9123456780",
  "createdAt": "2026-01-13T19:00:00",
  "updatedAt": "2026-01-13T19:00:00"
}
```

### 2. Get All Customers
```
GET /customers
```
**Response:**
```json
[
  {
    "customerId": 1234567890,
    "name": "Alice Smith",
    "email": "alice@example.com",
    "mobileNumber": "9123456780",
    "createdAt": "2026-01-13T19:00:00",
    "updatedAt": "2026-01-13T19:00:00"
  }
]
```

### 3. Get Customer by ID
```
GET /customers/{customerId}
```
**Example:**
```
GET /customers/1234567890
```
**Response:**
```json
{
  "customerId": 1234567890,
  "name": "Alice Smith",
  "email": "alice@example.com",
  "mobileNumber": "9123456780",
  "createdAt": "2026-01-13T19:00:00",
  "updatedAt": "2026-01-13T19:00:00"
}
```

### 4. Update Customer
```
PUT /customers/{customerId}
```
**Request Body:**
```json
{
  "name": "Alice Johnson",
  "email": "alicej@example.com",
  "mobileNumber": "9876543210"
}
```
**Response:**
```json
{
  "customerId": 1234567890,
  "name": "Alice Johnson",
  "email": "alicej@example.com",
  "mobileNumber": "9876543210",
  "createdAt": "2026-01-13T19:00:00",
  "updatedAt": "2026-01-13T19:05:00"
}
```

### 5. Delete Customer
```
DELETE /customers/{customerId}
```
**Example:**
```
DELETE /customers/1234567890
```
**Response:**
```
HTTP 204 No Content
```

## Account Endpoints

### 1. Create an Account
```
POST /accounts
```
**Request Body:**
```json
{
  "accountType": "SAVINGS",
  "branchAddress": "456 Park Avenue",
  "customerId": 1234567890
}
```
**Response:**
```json
{
  "accountNumber": 9876543210,
  "accountType": "SAVINGS",
  "branchAddress": "456 Park Avenue",
  "customer": {
    "customerId": 1234567890,
    "name": "Alice Smith",
    "email": "alice@example.com",
    "mobileNumber": "9123456780"
  },
  "createdAt": "2026-01-13T19:10:00",
  "updatedAt": "2026-01-13T19:10:00"
}
```

### 2. Get All Accounts
```
GET /accounts
```
**Response:**
```json
[
  {
    "accountNumber": 9876543210,
    "accountType": "SAVINGS",
    "branchAddress": "456 Park Avenue",
    "customer": {
      "customerId": 1234567890,
      "name": "Alice Smith"
    },
    "createdAt": "2026-01-13T19:10:00",
    "updatedAt": "2026-01-13T19:10:00"
  }
]
```

### 3. Get Account by ID
```
GET /accounts/{accountNumber}
```
**Example:**
```
GET /accounts/9876543210
```
**Response:**
```json
{
  "accountNumber": 9876543210,
  "accountType": "SAVINGS",
  "branchAddress": "456 Park Avenue",
  "customer": {
    "customerId": 1234567890,
    "name": "Alice Smith"
  },
  "createdAt": "2026-01-13T19:10:00",
  "updatedAt": "2026-01-13T19:10:00"
}
```

### 4. Update Account
```
PUT /accounts/{accountNumber}
```
**Request Body:**
```json
{
  "accountType": "CURRENT",
  "branchAddress": "789 Main Street"
}
```
**Response:**
```json
{
  "accountNumber": 9876543210,
  "accountType": "CURRENT",
  "branchAddress": "789 Main Street",
  "customer": {
    "customerId": 1234567890,
    "name": "Alice Smith"
  },
  "createdAt": "2026-01-13T19:10:00",
  "updatedAt": "2026-01-13T19:20:00"
}
```

### 5. Delete Account
```
DELETE /accounts/{accountNumber}
```
**Example:**
```
DELETE /accounts/9876543210
```
**Response:**
```
HTTP 204 No Content
```

---

## Notes
- IDs (`customerId`, `accountNumber`) are **10-digit numeric values generated by the application**.
- `createdAt` and `updatedAt` are automatically managed by the system.
- Use the above JSON payloads directly in **Postman** to test the APIs.
- `customer` inside `account` JSON will **not include `accounts`** to prevent circular references.

---

## Example Postman Collection
You can create requests in Postman using the above endpoints and request bodies. Set `Content-Type: application/json` in headers for POST/PUT requests.
